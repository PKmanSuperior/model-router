## ツールの概要

本プログラムは条件付き項書き換えシステム(CTRS)の実行不能性を検証するツールである. 
CTRSと実行不能性の検証対象となる式を入力として受け取り, 指定された実行時間内で実行不能性を検証する.
検証は関数Mainを通して実行され, 実行不能性を示すことが出来たときにtrueを返す.
実行時には検証の段階に対応したログが標準出力に表示される.
主な出力の意味は以下の通りである.

- `split:`  
  検証対象の式を分割する.

- `delete:`  
  不要な規則や条件を削除し，検証対象を簡約化する.

- `ordering check:`  
  書き換え規則が順序制約を満たすかどうかを確認する.
  `SUCCESS` は入力が項のサイズに関する順序を満たすことを表す.
  `FAILURE` は入力が項のサイズに関する順序を満たさないことを表す.

- `FOL encoding:`  
  CTRSと検証対象の式を第一階述語論理式に変換する.

- `sufficient condition check:`  
  十分条件に基づき，有限モデルが存在しないことを判定する.

- `model finding using Mace4/AGES:`  
  モデル生成器を用いてモデル探索を行う.
  `SUCCESS` はモデル生成に成功したことを表し, 下にモデルが表示される.
  `FAILURE` はモデル生成に失敗したことを表す.

---

## 入力について

本ツールでは，CTRSや検証対象の式を直接データ構造として与えるのではなく，入力ファイル名を文字列として受け取り，そこからCTRSと式の組を生成する関数readを用いる.

---

## 入力ファイルについて

入力ファイルは以下の要素から構成される:
- 関数記号とアリティ
- CTRSの書き換え規則
- 実行不能性の検証対象となる式

---

## 入力ファイルの例

```text
(fun a 0)
(fun b 0)
(fun f 1)
(rule (f x) x)
(rule (f a) b)
(infeasible? (= a x) (= b x))

---

## 実行例
1.SML/NJを起動する
2.以下を実行するプログラムをロードする
CM.make "sources.cm";
3.入力ファイルを読み込み, 検証を実行する.
val (ctrs, goals) = In.read "examples/ex1.in";
Main.main (ctrs, goals) 40;
